# image (Pug)

Pug-миксин, который используется везде, где нужно вставить изображение (в том числе картинки для соцсетей). Удобен в случаях, когда для разных устройств нужно отображать разные картинки.
Исключение: типовая страница, где используется просто тег img.

Расположение: `src/templates/shared/ui/image.pug`<br>
Аналоги: vue-компонент [X-Image](../components/ui/XImage.md)

## Аргументы

Аргументы передаются в объекте.

| Название                     | Тип       | По умолчанию | Описание                 |
|:-----------------------------|:----------|:-------------|:-------------------------|
| `src` <sup>*</sup>           | `string`  | -            | Путь к изображению   |
| `alt`                        | `string`  | alt          | Альтернативный текст изображения |
| `width` <sup>*</sup>         | `string`  | -            | Ширина изображения |
| `height` <sup>*</sup>        | `string`  | -            | Высота изображения |
| `lazy`                       | `boolean` | true         | Булево значение, указывающее, включена ли ленивая загрузка |
| `mode`                       | `string`  | -            | Класс модификатор, например: scale |
| `images`                     | `string`  | -            | Массив объектов, содержащих информацию в полях `src`, `mediaQuery` и `breakpoint` |

Поля объектов массива images:

| Название                     | Тип       | По умолчанию | Описание                 |
|:-----------------------------|:----------|:-------------|:-------------------------|
| `src` <sup>*</sup>           | `string`  | -            | Путь к изображению |
| `mediaQuery` <sup>*</sup>    | `string`  | max-width    | Медиа запрос mix-width или min-width |
| `breakpoint` <sup>*</sup>    | `string`  | -            | Значение медиа запроса |
| `width`                      | `string`  | -            | Ширина мобильного изображения |
| `height`                     | `string`  | -            | Высота мобильного изображения |

<sup>* - обязательный аргумент</sup>

## Использование и сниппеты

В шаблоне есть сниппеты, вставляющие заготовку кода pug. Можно использовать как сниппет, так и сам код.

Сниппет `img` вставляет в pug код:

```pug
+image(
	{
		src: 'img/',
		width: ,
		height: ,
	},
)
```

Сниппет `imgMedia` вставляет в pug код:

```pug
+image(
	{
		src: 'img/',
		width: ,
		height: ,
	},
	[
		{
			src: 'img/',
			mediaQuery: 'max-width',
			breakpoint: 768,
			width: ,
			height: ,
		},
	],
)
```
::: warning Важно
Нужно корректно заполнить все поля ```width``` и ```height```, так-как эти данные используются специалистами бэкенд-отдела для настройки ресайза.

Для мобильных изображений также важно прописывать эти размеры
:::

## Пример

```pug
+image(
	{
		src: 'img/content/banner.webp',
		width: 1440,
		height: 720,
		mode: 'main-banner'
	},
	[
		{
			src: 'img/content/banner-mobile.webp',
			mediaQuery: 'max-width',
			breakpoint: 480,
			width: 320,
			height: 540,
		},
		{
			src: 'img/content/banner-tablet.webp',
			mediaQuery: 'max-width',
			breakpoint: 768,
			width: 560,
			height: 720,
		},
	],
).main-hero-banner
```

Этот код вставит на страницу тег `<picture>`, который будет проверять медиазапросы и отображать одно из подходящих изображение: `banner-mobile.webp`, `banner-tablet.webp` или `banner.webp`. У добавленных этим миксином изображений есть стандартный для шаблона класс `img`, стили которого находятся в файле `/src/styles/shared/ui/img.sass`. Также миксин добавит картинке уникальные классы `main-banner` и `main-hero-banner` (оба способа из примера сработают, для чистоты кода лучше выбрать один из них).

Модифицировать стили изображения нужно по уникальному классу, перезаписывая значения свойств и переменных класса `img` (меняя размеры, соотношение сторон, скругление углов, object-fit и прочие свойства картинки).
