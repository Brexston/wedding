# 📄 project.config.ts

Файл служит единым источником конфигурации проекта.
Он описывает основные пути, параметры запуска и переменные окружения, чтобы использовать их в `vite.config.ts` и других
местах.

## Основные параметры

### Пути

- `SOURCE_DIR` — корневая папка исходников (`src`).
- `OUTPUT_DIR` — итоговая папка сборки для production (`../public/local/frontend-build`).
- `INITIAL_OUTPUT_DIR` — директория для первичной сборки (`dist`).
- `VIEWS_DIR` — директория с шаблонами страниц (`src/views`).

---

### Сервер

- `BASE_PORT = 8080` — порт для локальной разработки.

---

### Переменные окружения  
Загружаются через `loadEnv` из `.env` и могут быть переопределены:

- `PROJECT_NAME` — имя проекта (`VITE_APP_NAME` или имя текущей папки).
- `PROJECT_DOCS_NAME` — имя документации (`VITE_APP_DOCS_NAME` или имя папки).
- `PROJECT_TITLE` — заголовок проекта (`VITE_APP_TITLE` или `PROJECT_NAME`).
- `PROJECT_URL` — базовый URL для production (`VITE_APP_BASE_URL` или `/${PROJECT_NAME}/`).
- `PROJECT_INITIAL_URL` — базовый URL для initial-сборки (`VITE_APP_INITIAL_URL` или `/${PROJECT_NAME}/`).

## Зачем нужен

- Централизует все основные настройки проекта.
- Упрощает использование путей и переменных окружения в конфигурации Vite.
- Позволяет легко переключаться между разными режимами сборки и окружениями.
