# 📄 .editorconfig

Файл `.editorconfig` задаёт единые правила форматирования кода для всех участников проекта и разных редакторов/IDE.  
Он поддерживается большинством современных редакторов (VS Code, WebStorm, IntelliJ и др.) при наличии соответствующего
плагина.

## Что делает текущая конфигурация

- `charset = utf-8 `— кодировка файлов UTF-8.
- `end_of_line = crlf` — перевод строки в стиле Windows (CRLF).
- `indent_style = tab` — отступы с помощью табуляции.
- `tab_width = 4` — ширина табуляции в 4 пробела.
- `indent_size = unset` — явный размер отступа не фиксируется (берётся из `tab_width`).
- `insert_final_newline = true` — автоматически добавляет пустую строку в конце файла.
- `max_line_length = 120` — рекомендуемая максимальная длина строки.

## Настройки IntelliJ/WebStorm (префикс `ij_`)

- `ij_continuation_indent_size = unset` — размер продолженного отступа не фиксирован.
- `ij_formatter_tags_enabled = true` — поддержка специальных тегов для включения/выключения автоформатирования.
- `ij_smart_tabs = true` — «умные табы» (смешение табуляции и пробелов для выравнивания).
- `ij_wrap_on_typing = false` — отключена автоматическая переноска строк при вводе.

## Зачем нужен

`.editorconfig` помогает:

- избежать «войн стилей» в команде;
- держать единообразный формат кода вне зависимости от редактора;
- упростить ревью и уменьшить лишние диффы в Git.
