# 📁 config

Каталог с конфигурационными файлами, кастомными плагинами и вспомогательными утилитами **для сборки и разработки
проекта**. Всё, что находится здесь — относится не к бизнес-логике приложения, а к **инструментарию** (Vite, PostCSS,
Pug и пр.).

## Структура

### 📁 `helpers/`

* Общие вспомогательные функции для сборки, которые могут использоваться в конфиге Vite, PostCSS или других скриптах.
* Есть как `.js`, так и `.ts` версии index → это позволяет использовать утилиты и в чисто JS-скриптах, и в TypeScript-коде.

### 📁 `plugins/`

* Кастомные плагины для инструментов сборки.
* В текущем проекте — папка `pug/`, содержащая самописную надстройку над **Vite-плагином для работы с Pug-шаблонами**:
	- `vite-plugin-pug-build.ts` — логика для этапа **сборки** (build).
	- `vite-plugin-pug-serve.ts` — логика для этапа **разработки** (serve).
	- `vite-plugin-pug.ts` — общий экспорт/связка плагина.

Это нужно, так как стандартного плагина для Pug под Vite нет, и проект реализует собственное решение.

### 📁 `postcss/functions/`

* Набор кастомных `PostCSS-функций`, которые можно использовать в стилях.
* Реализованы утилиты для конвертации единиц измерения и адаптивной типографики:
	- `em/` → перевод значений в `em`.
	- `rem/` → перевод значений в `rem`.
	- `percent/` → работа с процентами.
	- `fluid/` → расчёт "fluid typography" или адаптивных размеров на основе max/min значений.
	- `roundNumber/` → округление чисел.
* Каждый модуль имеет:
	- `*.ts` — реализация функции.
	- `index.ts` — экспорт для удобного подключения в PostCSS.
* Центральный `index.ts` собирает все функции и передаёт их в конфиг PostCSS.
