# 📄 .env

Файл `.env` хранит переменные окружения для проекта.
Они автоматически подхватываются Vite и доступны в коде через `import.meta.env`.

## Текущие переменные

- `VITE_APP_TITLE` — человекочитаемый заголовок проекта (используется в шаблонах и мета-тегах).
- `VITE_APP_NAME` — системное имя проекта (применяется для путей, папок, выгрузки на xpager).
- `VITE_APP_DOCS_NAME` — имя документации (например, папка с документацией или URL).
- `VITE_APP_BASE_URL` — базовый путь для production-сборки.
- `VITE_APP_INITIAL_URL` — базовый путь для initial-сборки.

## Зачем нужен

- Позволяет конфигурировать проект без изменения исходного кода.
- Делает сборку гибкой для разных окружений (dev, staging, prod).
- Упрощает CI/CD — значения переменных можно переопределять на сервере.

::: tip Подсказка
Все переменные, которые должны быть доступны в коде клиента, обязательно начинаются с префикса `VITE_`.
:::
