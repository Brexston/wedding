# 📄 svgo.config.js

Конфигурация **SVGO** для оптимизации SVG-файлов в проекте.

## Основные задачи

* Оптимизация SVG без потери совместимости и семантики.
* Минификация и сортировка атрибутов для уменьшения размера.
* Сохранение нужных атрибутов и возможностей редактирования.

## Настройки

* **multipass: true**  
	Позволяет SVGO проходить по SVG несколько раз для максимальной оптимизации.
* **plugins** — список плагинов:
	1. **preset-default** с переопределениями:
       - `cleanupNumericValues: false` — не изменять точность числовых значений.
       - `removeViewBox: false` — сохранять атрибут `viewBox` (важно для масштабирования).
       - `cleanupIDs` — не удалять и не минифицировать ID.
       - `convertPathData: false` — не переписывать данные пути (чтобы не сломать сложные иконки).
    2. **sortAttrs** — сортирует атрибуты по алфавиту для консистентности.
    3. **addAttributesToSVGElement** — добавляет `xmlns="http://www.w3.org/2000/svg"` к корневому `<svg>` для корректного отображения в HTML.
    4. **removeDimensions** — убирает атрибуты `width` и `height`, оставляя только `viewBox`, что делает SVG масштабируемыми.

## Зачем нужен

* SVG остаются максимально легкими без потери визуального качества.
* Совместимость со всеми браузерами и компонентами Vue.
* Упрощает работу с иконками и спрайтами, используемыми в проекте.
