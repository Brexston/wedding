# 📁 .idea

Папка `.idea/` хранит настройки среды разработки [WebStorm](https://www.jetbrains.com/webstorm/) (и других IDE на базе
IntelliJ).
Она содержит проектные конфигурации, которые помогают разработчикам работать в едином окружении.

Основные элементы:

- `*.iml`, `modules.xml` — структура и модули проекта.
- `inspectionProfiles/` — правила инспекций кода (линтеры, подсветка ошибок).
- `jsLinters/`, `stylesheetLinters/` — интеграция с ESLint и Stylelint.
- `runConfigurations/` — готовые сценарии запуска (`dev`, `build`, `deploy` и т. д.).
- `watcherTasks.xml` — автоматические вотчеры (например, пересборка при изменении файлов).
- `shelf/` — временные сохранённые изменения (shelved changes).
- `workspace.xml` — личные настройки рабочей среды (расположение окон, вкладок и т. п.).

## Важный момент

Файлы в `.idea/` делятся на:

- **общие для команды** (например, линтеры, инспекции, runConfigurations) — их имеет смысл хранить в репозитории;
- **личные** (например, `workspace.xml`, `shelf/`) — обычно добавляются в .gitignore, чтобы не мешать коллегам.

Таким образом, `.idea/` нужна для унификации настроек IDE и удобства командной разработки, но требует внимательного
выбора того, что хранить в git.
