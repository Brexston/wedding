# 📁 .github

В шаблоне папка `.github/` используется для настройки и хранения GitHub-специфичных файлов.
Основной её сценарий — автоматизация процессов через [**GitHub Actions**](https://github.com/readme/guides/sothebys-github-actions).

В нашем случае внутри папки лежит каталог `workflows/`, где определяются CI-пайплайны.  
Файл `ci.yml` запускается автоматически при каждом push или pull request в ветку `main` и выполняет следующие задачи:

- Устанавливает Node.js и Bun в разных окружениях (Ubuntu, macOS).
- Скачивает зависимости проекта (`bun install`).
- Собирает проект (`bun run build`)

Таким образом, `.github/workflows/ci.yml` гарантирует, что проект корректно собирается и тестируется на разных ОС и
версиях Node.js, а ошибки будут обнаружены до попадания кода в основную ветку.
