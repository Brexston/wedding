# 📄 stylelint.config.js

Конфигурация **Stylelint** для проекта, включающая правила для Sass, Vue и HTML, а также поддерживающая порядок CSS-свойств и BEM-подобные классы.

## Основные задачи

* Проверка стилей на соответствие стандартам проекта.
* Поддержка **Sass (SugarSS)** и встроенных стилей в `.vue` и `.html`.
* Организация CSS/SCSS-свойств в логические группы для читаемости.
* Поддержка плагинов `stylelint-order` и `@stylistic/stylelint-config` для единообразного код-стиля.

## Структура

- **extends** — подключение базовых конфигураций:
  * `@stylistic/stylelint-config` — основной код-стиль.
  * `stylelint-config-html/vue` — поддержка Vue и HTML.
  * `stylelint-config-recommended-scss` — рекомендуемые правила SCSS.
  * `stylelint-config-recommended-vue/scss` — рекомендации для Vue + SCSS.
* **plugins** — `stylelint-order` для контроля порядка CSS-свойств.
* **overrides** — отдельные синтаксисы для разных типов файлов:
  * `*.sass` — SugarSS.
  * `*.vue` и `*.html` — через `postcss-html`.
* **rules** — сотни правил, включая:
  * Порядок правил и селекторов (`order/order`, `order/properties-order`).
  * Стиль комментариев и форматирование (`comment-empty-line-before`, `declaration-empty-line-before`).
  * Поддержка SCSS-специфичных правил (`scss/dollar-variable-*`, `scss/at-*`).
  * Проверка на использование вендорных префиксов, дублей и неизвестных свойств.
  * Частично поддержка BEM-паттерна (закомментировано, можно включить при желании).
* **ignoreFiles** — исключает ненужные файлы и директории (`node_modules`, `dist`, `.git`, .`vscode`) и оставляет только исходные Sass/Vue файлы.
* **fix: true** — позволяет автоматически исправлять ошибки при запуске Stylelint.

## Зачем нужен

* Поддерживает единый стиль CSS/SCSS во всем проекте.
* Минимизирует ошибки и конфликты при командной разработке.
* Облегчает рефакторинг и читаемость кода.
* Интегрируется в CI/CD через `npm run lint`.
