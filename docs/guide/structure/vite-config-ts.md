# 📄 vite.config.ts

Файл конфигурации [Vite](https://vite.dev/) управляет процессом сборки, разработкой и оптимизацией проекта.  
Здесь задаются пути, плагины, PostCSS и Sass-настройки, алиасы, а также логика для разных режимов запуска.

## Основные особенности

### Режимы

В проекте используются кастомные режимы:

- `development` — локальная разработка.
- `production` — финальная сборка.
- `initial` — первичная сборка.
- `deploy` — деплой вёрстки на xpager.
- `watch` — отслеживание файлов при разработке.

От режима зависят `BASE_URL`, выходные директории и минимизация.

---

### Сервер разработки

- Запуск на `0.0.0.0` (доступ с локальной сети).
- Порт задаётся через `BASE_PORT`.
- Есть заготовка для проксирования API.

---

### Сборка

- Разделение чанков и ресурсов по папкам (`scripts`, `styles`, `fonts`, `img`, `videos`).
- Использование `terser` и `lightningcss` для минификации (отключается в watch-режиме).
- Поддержка манифеста в production.

---

### CSS и PostCSS

- Автопрефиксер.
- `pxtorem` для адаптивных размеров.
- Кастомные PostCSS-функции (`fluid`, `em`, `percent`).
- Sass с глобальными переменными и миксинами.

---

### Алиасы

- `@` → `src`
- `@app` → `src/app`
- `@styles` → `src/styles`
- `@img` → `src/assets/img`

---

### Подключённые плагины

- `@vitejs/plugin-vue` — поддержка Vue.
- `vite-plugin-pug` — шаблоны Pug с передачей локалей.
- `unplugin-fonts` — подключение кастомных шрифтов.
- `vite-plugin-juice` — inlining стилей для email-шаблонов.
- `vite-plugin-svg-spritemap` — генерация SVG-спрайта.

## Зачем нужен

Конфигурация `vite.config.ts` формирует стандартизированную среду разработки и сборки, позволяя:

- поддерживать разные режимы работы;
- единообразно обрабатывать шрифты, иконки, стили и скрипты;
- оптимизировать код под production;
- использовать удобные плагины для Vue, Pug, email-шаблонов и SVG.
