# 📁 docs

Папка `docs/` — это **документация шаблона**.

Она нужна для того, чтобы:

* **Объяснять устройство проекта** — какие есть папки, конфиги, инструменты и зачем они нужны (`structure/`).
* **Документировать внутренние решения** — собственные компоненты, хуки (`composables/`), mixins для Pug и Sass, общие
  UI-паттерны.
* **Давать инструкции по запуску и использованию** — как стартовать проект, какие команды есть, как подключать API,
  шрифты, капчу и т. д. (`introduction/`, `essentials/`).
* **Демонстрировать примеры** — вместе с markdown-файлами лежат Vue-компоненты (`useI18nExample.vue`,
  `ViewportExample.vue`), а также картинки и скриншоты для гайдов (`docs/public`).
* **Служить справочником по UI** — есть отдельный раздел для всех универсальных компонентов (например, `XInput*`,
  `XSlider*`, `XModal`), чтобы разработчики знали их API и варианты использования.
* **Обеспечивать консистентность команды** — новые участники быстро погружаются в проект, а текущие всегда имеют под
  рукой "источник истины".

Фактически `docs/` — это локальный **Storybook + вики проекта в markdown**, который документирует всё: от конфигов и
инфраструктуры до компонентов и пользовательских функций.
