# 📄 tsconfig.node.json

Конфигурация TypeScript для **Node.js-части проекта**, например скриптов сборки и конфигурации Vite.

## Основные настройки

* **compilerOptions** — ключевые параметры компилятора:
  - `"composite": true` — разрешает использование проекта как композитного (поддержка references в tsconfig.json).
  - `"module": "ESNext"` — использование современных ECMAScript модулей для Node.js.
  - `"moduleResolution": "node"` — стандартное разрешение модулей Node.js.
  - `"allowSyntheticDefaultImports": true` — разрешение синтетических default-импортов для CommonJS модулей.
  - `"skipLibCheck": true` — пропуск проверки типов для библиотек в `node_modules` (ускоряет компиляцию).
* include — файлы, которые компилируются:
	```json
	[
		"./*.ts",
		"./config/**/*.ts"
	]
	```
* exclude — файлы и папки, исключаемые из компиляции:
	```json
	[
		"node_modules",
		"dist"
	]
	```

## Зачем нужен

- Отдельная конфигурация для Node.js кода, чтобы не мешать клиентскому `tsconfig.json`.
- Позволяет использовать современные модули и типизацию при работе с инструментами сборки и скриптами.
- Обеспечивает корректную работу ссылок (`references`) между проектами TypeScript.
