# Страницы

После запуска проекта (на этапе вёрстки) в шаблоне фронтенда уже присутствуют несколько [стандартных страниц](https://html.xpager.ru/frontend-template/), на основе которых строится будующий проект.

![Шаблон ронтенда](/img/content/frontend-template.webp)

- [Главная страница](https://html.xpager.ru/frontend-template/views/main.html) - содержит заготовки шапки, подвала и простой контент для отображения

- [Корзина (пустая)](https://html.xpager.ru/frontend-template/views/cart-empty.html) и [Корзина](https://html.xpager.ru/frontend-template/views/cart.html) - шаблонные решения корзины, подготовленные для взаимодействия с бэком

- [Медиа](https://html.xpager.ru/frontend-template/views/media.html) - страница с однотипными элементами и пагинацией; подходит для новостей, товаров и других элементов, приходящих с бэка

- [Шаблон письма](https://html.xpager.ru/frontend-template/views/emails/main.html) - содержит вёрстку для бэка, которая используется для отправки email из Битрикса

- [404](https://html.xpager.ru/frontend-template/views/404.html) и [502](https://html.xpager.ru/frontend-template/views/502.html) ошибки... тут нечего добавить 🙆

- [UI Элементы](https://html.xpager.ru/frontend-template/views/ui.html) - все элементы, которые будут использоваться на проекте (кнопки, инпуты, модалки, формы и так далее); ui-kit переносится из макета в самом начале вёрстки, поэтому если в нём нет какого-то элемента, но он есть на макете, то нужно серьёзно поговорить с дизайнером ✊🏼

- [Типовая страница](https://html.xpager.ru/frontend-template/views/content.html) содержит элементы (div, p, ul и т.д.) без классов; её стилизуют для того, чтобы контент, который менеджеры добавляют через админку Битрикса, соответствовал общему стилю проекта

- [Модальные окна](https://html.xpager.ru/frontend-template/views/modals.html) - техническая страница для тестировщиков, на которой собираются все модалки, которые есть на проекте (изначально наполнена шаблонным контентом, который нужно заменить)
